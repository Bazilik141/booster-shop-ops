# Codex Report — CRM-002: статус `На складі UA` не оприбутковує лоти

Date: 2026-06-27

## Scope

Діагностика та виправлення посилки `LX322066112JP` у живій Google Sheets CRM.
Оновлено формули вкладки `Склад` та source-copy Apps Script. Дані закупок,
продажів і статуси трьох лотів не змінювалися.

## Evidence

У `Закупки` посилка містить три лоти з датою доставки `2026-06-27` і статусом
`На складі UA`:

- row 44, `LOT-0048`, `YGO-JP-BDOM-BST`, 30 од.;
- row 58, `LOT-0063`, `OP-JP-OP15-BBX`, 2 од.;
- row 62, `LOT-0067`, `PKM-JP-SVEX-BLR`, 1 од.

У `Склад` ці кількості не потрапили в колонку `Закуплено`:

- row 38, `YGO-JP-BDOM-BST`: закуплено 0, закуплено всього 30;
- row 37, `OP-JP-OP15-BBX`: закуплено 0, закуплено всього 2;
- row 45, `PKM-JP-SVEX-BLR`: закуплено 0, закуплено всього 4; 3 старі одиниці
  залишаються у `Очікується / Японія`.

## Root cause

Формули `Склад!E:E` включають лише статуси:

`На складі`, `Частково продано`, `Продано`.

Статус `На складі UA` відсутній, тому нові лоти дають 0 у `Закуплено` і
`Залишок`.

Той самий drift є в живій source-copy `Apps_Script_код`:

- `getFifoCostBatches_()` не включає `На складі UA`;
- `updateLotStatuses()` не включає `На складі UA` ні в `updatable`, ні в
  `allowedForFifo`;
- `apiSummary_()` уже включає `На складі UA`, тому різні частини CRM
  класифікують той самий статус по-різному.

## Risk

Поки статус лишається `На складі UA`:

- фізичний залишок у `Склад` занижений на 33 одиниці;
- нові лоти не беруть участі у FIFO-собівартості продажів;
- `updateLotStatuses()` не нормалізує та не оновлює ці лоти;
- summary/API може показувати вартість складу, яка не узгоджується з кількістю
  у вкладці `Склад`.

## Implemented

1. У `Склад!E:E` додано `На складі UA` до дозволених складських статусів.
   Оновлено 199 формул: один тестовий рядок + 198 формул одним bounded
   `findReplace`.
2. У `Apps_Script_код!A1507` додано `На складі UA` до
   `getFifoCostBatches_()`.
3. У `Apps_Script_код!A2121:A2122` додано `На складі UA` до
   `updatable` та `allowedForFifo`.
4. У `Apps_Script_код!A2143` збережено канонічний статус непроданого нового
   лота: `На складі UA`; старі лоти `На складі` не мігрують автоматично.
5. Додано reviewable replacement:
   `patches/CRM-002_ua-warehouse-status_20260627.js`.

Зміни внесені адресно у чотири рядки source-copy. Доданий Claude код у кінці
Apps Script та функціонал Telegram-бота не перезаписувалися і не змінювалися.

## Deployment boundary

`Склад!E:E` виправлено безпосередньо у живій CRM і перераховано.
`Apps_Script_код` — лише source-copy; прив'язаний Apps Script не оновлено,
оскільки прямого Apps Script API/clasp немає, а Chrome-сесія недоступна.
До копіювання цих змін у bound Apps Script нові лоти вже відображаються у
`Склад`, але live FIFO-функція може ще не бачити `На складі UA`.

## Verification

- Live bounded reads: `Закупки!A1:T309`, `Склад!A1:T220`.
- Formula reads: `Склад!E37:E38`, `Склад!E45`.
- Apps Script source-copy reads: `Apps_Script_код!A1503:A1515`,
  `A1797:A1802`, `A2117:A2154`.
- Formula smoke test: `OP-JP-OP15-BBX` змінився з 0 на 2 до rollout.
- Rollout: 198 додаткових формул оновлено, без помилок batch update.
- Final readback:
  - `OP-JP-OP15-BBX`: закуплено/залишок = 2/2;
  - `YGO-JP-BDOM-BST`: закуплено/залишок = 30/30;
  - `PKM-JP-SVEX-BLR`: закуплено/залишок = 1/1.
- Source-copy readback: усі чотири змінені рядки збігаються з очікуваними.
- Patch syntax: `node --check` — OK.
- Bound Apps Script не компілювався і не запускався.
- Idempotency: повторний formula `findReplace` змінить 0 входжень.

## Rollback

Для формул: прибрати додану умову
`Закупки!Q="На складі UA"` з `Склад!E:E`.

Для source-copy: повернути попередні значення в
`Apps_Script_код!A1507`, `A2121:A2122`, `A2143`.
