# Claude Report — RD-13: checkout reskin round 6

Date: 2026-07-08
Патч: `patches/RD-13_checkout-reskin-round6_20260708.php` (4 файли: twig, css, js + **1 рядок у контролері** — погоджено нижче)

## 1. Short conclusion — знайдено системний корінь

Живий дебаг показав: у checkout.twig працює ВЛАСНИЙ MutationObserver
(`enhanceCheckout`, дебаунс 40мс), який у своєму циклі безумовно переписує
текст legend-ів (`.text()` створює новий текстовий вузол навіть із тим самим
текстом → нова мутація → знову цикл). Мій планувальник був "clear+set"
дебаунсом — потік їхніх мутацій скидав мій таймер до нескінченності, і після
першого ж запуску вся синхронізація голодувала. Саме тому: зникав підтекст
"Безпечно через еквайринг" після кліку, прев'ю-методи лишались без
relabel, relabel'и "вмирали" в рандомні моменти. **Фікс: планувальник став
троттлом — таймер, що очікує, більше не скидається; sync гарантовано
виконується ≤60мс після першої мутації пачки.**

## 2. Класифікація правок (як просив)

СПІЛЬНІ (одна правка — обидва формати):
- Троттл-фікс синхронізації (корінь для підтексту оплати і мертвих relabel).
- Радіо "Я хочу використовувати існуючу адресу"/"Додати нову адресу" —
  сховані (перемикання лише лінками "+ Інша адреса"/"← Використати
  збережену"). Інпути в DOM, логіка модуля незмінна.
- Шеврон — праворуч, на одному місці у всіх картках.
- Текст згорнутих блоків: 13.5px/600, темніший — читається без мруження.
- Збережені адреси у select: короткий формат "Дніпро, Поштомат №49489" /
  "Дніпро, Відділення №22" / "Дніпро, вул. Савкіна" (повний текст
  зберігається в data-атрибуті і показується в recap Отримувача).
- Recap Отримувача: "ПІБ · телефон" + повна адреса (телефон — див. п.4).
- Прибрано "Captcha"-legend і порожнє місце: капча тепер забирається цілим
  fieldset-ом (із запобіжником), залишкові написи/порожні row ховаються.

ДЕСКТОП:
- "Замовлення" не згортається — шеврон прибраний (≥901px).

МОБАЙЛ:
- Overflow-фікс: `.bs-co-grid` і всі картки обмежені шириною екрана
  (min-width:0 — грід-елементи більше не розпирає широкий вміст).
- Гість: промокод переїжджає в нижній блок (Промокод → Коментар → Оферта →
  Реєстрація → CTA → Капча → Новини). Авторизований: промокод лишається в
  згорнутому "Замовленні" (як затверджено раніше).
- Капча: нативний розмір (transform:none, контейнер на всю ширину).

ГІСТЬ (обидва формати):
- Телефон і E-Mail фізично переносяться в сітку Отримувача (вони жили в
  ОКРЕМОМУ .row — тому ширини і позиція не слухались) → телефон праворуч
  від "По батькові", однакові ширини.
- Кнопка не активувалась після вибору відділення/поштомату: вибір пункту з
  дропдауна модуля тепер повторно генерує штатні change+blur — існуючий
  autosave-шлях ST-2a.8d спрацьовує без "введи ще щось і натисни ентер".
  (Це твій "костиль", реалізований через стандартні події, без нових
  ендпоінтів.)

## 3. Про polyfill.js помилки в консолі

`Extension context invalidated` — це НЕ сайт. Це контент-скрипт розширення
Chrome (перекладач/тощо), яке перезавантажилось при відкритому табі. На
роботу checkout не впливає, у клієнтів без цього розширення не з'являється.

## 4. Контролер — 1 рядок (потребує твого ОК фактом деплою)

Телефону авторизованого клієнта фізично немає в DOM — жодна презентаційна
магія його не дістане. За твоїм повторним запитом у патч включено
мінімальну read-only зміну `catalog/controller/checkout/checkout.php`:
`$data['customer_telephone'] = $this->customer->getTelephone()` (тільки для
залогінених) + вивід у data-атрибут кореневого div. Жодної логіки оплат/
доставки/цін не торкається. Якщо НЕ хочеш чіпати контролер — скажи, зберу
r6 без цього (recap буде без телефону).
