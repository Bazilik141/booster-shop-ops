# RD-13.1D — checkout visual root fixes

## Scope

Visual checkout cleanup based on `booster-rd13-visual-current.tar.gz` supplied by
the owner. No DB, checkout route, payment, order-creation, or server CAPTCHA
validation changes.

## Changes

- removes normal autosave status blocks at their source; an error status is
  created only when a real `register.save` error occurs;
- keeps the real CAPTCHA in the sidebar by moving only the native
  `#g-recaptcha-*` fieldset, never a generic iframe or Bootstrap column;
- removes CSS iframe scaling and the broad `:has()` hiding workaround;
- uses Google's supported `compact` CAPTCHA size only for mobile checkout
  widths up to 420px; desktop keeps the configured native size;
- resets the moved CAPTCHA's Bootstrap row gutters, preventing invisible
  horizontal geometry from escaping the card;
- defines one desktop grid source: phone is column 2, email is column 1;
  both return to normal single-column placement on mobile.
