# Claude Report — RD-13: checkout reskin round 5 (polish)

Date: 2026-07-07
Патч: `patches/RD-13_checkout-reskin-round5_20260707.php`

## 1. Short conclusion

Полірувальний прохід по ревю r4. Найважливіше — знайдений корінь
"дублюючого відображення товарів": на гостьовому флоу виняток усередині
sync() лишав прапорець `syncing=true`, і вся презентаційна синхронізація
вмирала — сирий кеш таблиці confirm залишався видимим. Виправлено двома
рівнями: (а) sync() тепер ізолює кожен крок (try/catch + finally) — одна
помилка не вбиває конвеєр; (б) детермінована страховка — кеш-обгортка в
checkout.twig отримала клас `bs-co-src-table`, і CSS ховає сиру таблицю
безумовно, незалежно від стану JS.

## 2. Зміни по пунктах ревю

Десктоп:
- "Авторизуйся" без жирного (це не кнопка).
- Коментар: 2 рядки за замовчуванням, авто-ріст по контенту (до 300px).
- Дубль товарів після згоди з офертою — усунено (див. вище).
- Телефон у сітці Отримувача: порядок тепер проставляється inline-стилями
  на кожну дитину сітки (CSS-класовий порядок був крихким проти стокових
  блоків без класів); min-width:0 проти розпирання колонки.
- Залишковий напис "Captcha" в Отримувачі ховається; блок капчі забирається
  цілим .col-контейнером.
- Плитки типу доставки: відділення → поштомат → курʼєр.
- Прогрес-бар безкоштовної доставки рендериться завжди (до вибору доставки
  ціна показується як "—", прогрес від "До сплати").
- Замовлення: сума в шапці і рядок "Сума товарів" приховані на десктопі
  (на мобайлі лишаються — так у мокапі); шапка 16px/800.
- Кнопка "Застосувати": синя обводка + синій текст.
- Згортання/розгортання Отримувача і Доставки працює й на десктопі.

Мобайл:
- Шеврони: SVG замість текстового символу, правильна орієнтація
  (розгорнуто = вгору, згорнуто = вниз), плавний поворот.
- Типографіка згорнутих блоків: назва 15px/700 ink, підрядок 12.5px
  regular ink-3 — по хендофу.
- Recap Отримувача: ім'я + адреса замість фрази "Дані з вашого акаунта…".

  продом по HTTP: JS `0e4688f3…` і CSS `bdf8e5a8…` збігаються байт-у-байт.
- php-parser: 0 помилок; node --check OK; симуляція всіх анкерів OK.
- Той самий контракт: ідемпотентність, dry-run, DIR_CACHE, self-delete,
  бекапи, відкат.
