## Технічна база

- OpenCart 4, сервер LiteSpeed. БД-префікс таблиць `ocp5_`. Українська = `language_id 4`.
- Описи в БД зберігаються entity-encoded (htmlspecialchars: `&lt;p&gt;`).
- **GOTCHA OpenCart 4:** catalog-шаблони рендеряться через `ArrayLoader` — `{% include 'xxx.twig' %}` сторонніх partial-файлів **НЕ працює** (`Template is not defined`, 500 на проді). Реюзабельні asset-и (CSS/JS) треба інлайнити в шаблон або вантажити через дочірній контролер, не через `{% include %}`.
- Атрибути товарів `attribute_id`: 12=Мова, 13=Назва сету, 14=Рік, 15=Карток у бустері, 16=Бустерів у боксі, 17=Стан, 18=Походження, 19=Зважування, 20=Виробник, 21=Тип пакування.

## Метод рев'ю патчів Codex

1. Розпакувати потрібні файли + БД-дамп з бекапу в пісочницю.
2. Звірити assert-значення / replaceOnce-цілі / факти патчу з реальним станом.
3. Перевірити scope (нічого зайвого), forbidden-claims, do-not-touch зони, rollback.
4. Дати owner-чеклист на live-QA. Статус «Готово» — лише після live-QA власника.

## Жорсткі правила

- Не чіпати без явної потреби й окремого QA: sitemap.xml, robots.txt, .htaccess, redirects, canonical, schema/JSON-LD, Merchant feed.
- checkout / payment / Hutko / Checkbox / fiscalization — high-risk, завжди явно називати ризик + smoke-test.
- Зелений колір — тільки purchase actions (купити, кошик, success).
- Не вигадувати GTIN, reviews, aggregateRating. Не робити окремий HTML-блок «Походження та чесність продажу» в описах. FAQPage JSON-LD поки не додавати.
- Тон контенту: українською, прямо, без води, без keyword stuffing, без фальшивих гарантій.
- Якщо задача має ризик зламати сайт/checkout/payment/sitemap/robots/canonical/Merchant feed/CRM — явно назвати ризик і дати smoke-test.
