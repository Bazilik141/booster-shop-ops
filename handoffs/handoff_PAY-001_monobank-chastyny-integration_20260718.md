У наданому фрагменті API-докс приклад тіла `/api/order/create` містить лише `store_order_id`, `client_phone`, `total_sum` — параметра кількості платежів там немає. ## 7. Довідка — API monobank «Покупка Частинами» (витяг з наданої докс, для самодостатності хендофа)

### 7.A Автентифікація
Кожен запит — заголовки `store-id` (ідентифікатор магазину) і `signature`.
`signature = Base64(HMAC-SHA256(request_body_bytes, store_secret))`

```php
function calculateSignature($storeSecret, $requestBody) {
    return base64_encode(hash_hmac('sha256', $requestBody, $storeSecret, true));
}
// приклад тіла: {"store_order_id":"ORD-001","client_phone":"+380501234567","total_sum":1500.00}
```

### 7.B Флоу інтеграції
1. Клієнт обирає товари й оплату через ПЧ.
2. Магазин: `POST /api/order/create` (опційно `result_callback`).
3. Клієнт отримує push у застосунку monobank, підтверджує договір ПЧ.
4. Статус → `IN_PROCESS/WAITING_FOR_STORE_CONFIRM`.
5. Якщо задано `result_callback` — monobank сам POST-ить на нього; інакше — магазин опитує `POST /api/order/state`.
6. Магазин видає товар → `POST /api/order/confirm` (або `POST /api/order/reject` при відмові у видачі).

### 7.C Callback
Надсилається **лише у 2 випадках**: перехід у `IN_PROCESS/WAITING_FOR_STORE_CONFIRM`, і перехід у будь-який `FAIL/*`. Для інших станів (напр. `IN_PROCESS/WAITING_FOR_CLIENT`) callback НЕ надсилається — обов'язковий polling через `/api/order/state`.

Формат:
```json
{"order_id": "fa4a8249-336e-4e6d-9b85-79bc8be62377", "state": "IN_PROCESS", "order_sub_state": "WAITING_FOR_STORE_CONFIRM"}
```
Безпека: перерахувати підпис тим самим методом, порівняти із заголовком `signature` callback-запиту, обробляти лише при збігу.

### 7.D Довідник статусів
| Статус | Підстатус | Опис | Дія |
|---|---|---|---|
| SUCCESS | ACTIVE | Товар видано, ПЧ активна | Фінальний, дій не треба |
| SUCCESS | DONE | ПЧ повністю сплачена | Фінальний |
| SUCCESS | RETURNED | Товар повернено, гроші клієнту | Фінальний |
| IN_PROCESS | WAITING_FOR_CLIENT | Очікує підтвердження клієнта в застосунку | Чекати, callback не приходить — polling |
| IN_PROCESS | **WAITING_FOR_STORE_CONFIRM** | Клієнт підтвердив — можна видавати товар | Видати товар + `/api/order/confirm` |
| FAIL | CLIENT_NOT_FOUND | Не клієнт monobank / не знайдено | Перевірити формат телефону |
| FAIL | EXCEEDED_SUM_LIMIT | Перевищено ліміт ПЧ клієнта | Зменшити суму / інший спосіб оплати |
| FAIL | EXISTS_OTHER_OPEN_ORDER | У клієнта вже є відкрите замовлення ПЧ | Почекати 15 хв або скасувати старе |
| FAIL | NOT_ENOUGH_MONEY_FOR_INIT_DEBIT | Не вистачає коштів на перший платіж | Клієнт поповнює картку |
| FAIL | REJECTED_BY_CLIENT | Клієнт відхилив ПЧ | Інший спосіб оплати |

### 7.F Обробка помилок
HTTP: `200` успіх · `201` замовлення створено · `400` bad request (валідація) · `401` підпис відсутній/невалідний · `403` заборонено · `404` не знайдено · `429` rate limit · `500` внутрішня помилка.
Формат помилки: `{"message": "текст помилки"}`.

### 7.G Best practices (з докс)
Валідувати дані клієнта до відправки; унікальний `store_order_id`; завжди вказувати `result_callback`; обробляти ідемпотентність; негайно обробляти `WAITING_FOR_STORE_CONFIRM`; перевіряти підпис callback; мати fallback-polling; логувати всі зміни статусу; повторювати лише 5xx з експоненційним backoff; при зверненні в підтримку — додавати `Trace-Id`, Store ID, Order ID, приклади запитів/відповідей, timestamps.

### 7.H Бренд/naming (з файлу «Про бренд»)
Дозволені написання: «Покупка Частинами», «Покупка Частинами monobank», «Покупка Частинами monobank | Universal Bank». **Не писати:** «Оплата частинами monobank Монобанк», «АТ Universal Bank», «Кредит monobank». Позиціонування — «безкоштовна розстрочка», обов'язково комунікувати клієнту, що сервіс надається за рахунок магазину. Емітент: АТ «Універсал Банк», ліцензія НБУ №92 від 20.01.1994, № 226 у держреєстрі банків, учасник Фонду гарантування вкладів. Підключення: Telegram-бот `@chast_monobankbot` → «Стати партнером» → анкета → API (потрібен програміст) або через бот (покрокова інструкція) → ~3 дні. Повна тарифна сітка — `monobank.ua/chast/vendors` (не в наданих файлах — не вигадувати числа поза прикладом 2,9%/2 платежі).
