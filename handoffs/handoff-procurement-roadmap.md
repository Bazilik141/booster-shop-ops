# Codex Handoff — Вкладка Закупівельний_Роадмап у CRM

## 1. Task ID
CRM-ROADMAP-001

## 2. Context

**CRM:** `1PvlSlg3UoPw8Fbj98lHL-VGLB0HP8hgKUxsXPW1GkRg`

Зараз у CRM немає місця для планування майбутніх закупок. Власник приймає рішення щодо нових товарів ad hoc. Потрібна вкладка де можна фіксувати потенційні SKU, рахувати прогнозовану маржу і бачити пріоритет.

Ця вкладка — **тільки ручне введення власником** + формульні розрахунки. Ніяких Apps Script тригерів, ніякого автозапису.

## 3. Goal

Додати вкладку `Закупівельний_Роадмап` у CRM spreadsheet з такою структурою:
- Заголовок і легенда зверху
- Основна таблиця для введення даних
- Формульні колонки: прогнозована маржа, гранична ціна закупки, рекомендація
- Умовне форматування: підсвічувати пріоритет і статус

## 4. What to change

### 4.1 Нова вкладка `Закупівельний_Роадмап`

**Розмістити:** після вкладки `Закупки`, перед `Продажі` (або в кінці — на розсуд Codex після перевірки порядку вкладок).

### 4.2 Структура таблиці

**Рядок 1:** Заголовок "Booster Shop — Закупівельний роадмап"  
**Рядок 2:** Примітка: "Ручне введення. Формули у виділених колонках не редагувати."  
**Рядок 3:** Заголовки колонок  
**Рядки 4+:** Дані

**Колонки (A–N):**

| Кол. | Назва | Тип | Примітка |
|------|-------|-----|---------|
| A | ID | Авто/ручне | RDMP-0001, RDMP-0002... або пусто |
| B | SKU (якщо вже є в CRM) | Ручне | Посилання на Товари якщо є |
| C | Назва товару | Ручне | Повна або коротка |
| D | Бренд | Ручне | Pokémon / One Piece / YGO / MTG |
| E | Формат | Ручне | Booster / Booster Box / Mystery Box / інше |
| F | Джерело закупки | Ручне | ZenMarket / прямий постачальник / OLX / інше |
| G | Ціна закупки (JPY або грн) | Ручне | Сума за одиницю |
| H | Курс JPY (якщо JPY) | Формула | =IFERROR(VLOOKUP("JPY";Курси!A:B;2;FALSE);3.5) |
| I | Ціна закупки (грн) | Формула | =IF(G4="";"";IF(F4="ZenMarket";G4/H4;G4)) |
| J | Очікувана ціна продажу (грн) | Ручне | РРЦ або цільова ціна |
| K | Прогнозована маржа % | Формула | =IF(OR(I4="";J4="");"";(J4-I4)/J4) |
| L | Гранична закупка (грн) | Формула | =IF(OR(I4="";J4="");"";J4*0.65) — 35% для ходових |
| M | Пріоритет | Ручне | Dropdown: Висока / Середня / Низька |
| N | Статус | Ручне | Dropdown: Ідея / Розглядається / Заплановано / Куплено / Відхилено |
| O | Ризик | Ручне | Текст: наприклад "низький попит", "курсовий ризик" |
| P | Нотатки | Ручне | Будь-що |

**Примітка щодо колонки L (гранична закупка):**
- Поточна логіка в CRM: 35% маржа для ходових SKU, 40% для повільних
- Для роадмапу — базова формула 35% (власник коригує вручну якщо потрібно)
- Формула: `=IF(OR(I4="";J4="");"";ROUND(J4*0.65;0))`
