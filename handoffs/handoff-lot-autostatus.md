**File:** `Code.gs` в Apps Script проекті CRM.

**Проблема:** Статус лотів у вкладці `Закупки` оновлюється вручну. Якщо куплено 2 партії Mega Brave по 30 шт (лот 1 і лот 2), продано 32 шт — лот 1 фактично вичерпаний, але в таблиці досі "На складі". Власник не бачить реального стану.

**Поточні статуси у Закупки (колонка Q, індекс 16):**
- `На складі` — товар прийшов і є в наявності
- `Частково продано` — продано частину лоту
- `Продано` — лот вичерпаний
- `В дорозі`, `На складі в Японії`, `Виграно` — товар ще не в Україні, **не чіпати**

**Як рахується FIFO у Code.gs:**
Функція `getFifoCostBatches_(ss, sku, saleDate)` вже збирає всі лоти по SKU в хронологічному порядку. Функція `getConsumedQtyBeforeSale_` рахує скільки продано до певного продажу. Ця ж логіка потрібна для розрахунку залишку по кожному лоту.

**Ключові колонки Закупки:**
- A = LOT-ID (LOT-0001, LOT-0002, ...)
- B = ZenMarket Order №
- C = Трек-номер
- D = Дата доставки в Україну
- E = SKU
- H = Кількість одиниць
- Q (індекс 16) = Статус

## 3. Goal

Додати функцію `updateLotStatuses()` в `Code.gs`, яка:
1. Для кожного лоту зі статусом `На складі` або `Частково продано` рахує скільки одиниць з цього лоту фактично продано (FIFO-логіка)
2. Оновлює статус:
   - продано = 0 → `На складі`
   - 0 < продано < кількість лоту → `Частково продано`
   - продано ≥ кількість лоту → `Продано`
3. Додає кнопку в меню Booster CRM для ручного запуску
4. Додає тригер для автозапуску щодня або після додавання продажу (на розсуд Codex, але не частіше 1x/день)

**Алгоритм FIFO для лотів:**

Для кожного унікального SKU в Закупки:
1. Зібрати всі лоти по цьому SKU з дозволеними статусами (+ `Продано`), відсортувати за датою прийому (колонка D) → якщо дата відсутня — лот ігнорувати або ставити в кінець
2. Порахувати загальну кількість продажів по цьому SKU з вкладки `Продажі` (тільки фактичні продажі — `isActualSaleForCost_` вже є)
3. Розподілити продажі FIFO: лот 1 → лот 2 → лот 3:
   - якщо продано ≥ кількість лоту 1 → лот 1 отримує "Продано", залишок іде далі
   - якщо продано < кількість поточного лоту → "Частково продано"
   - якщо до лоту продажів ще не дійшло → "На складі"
4. Записати нові статуси тільки там де вони змінились

**Важливо:** не рахувати списання (Списання) у визначенні статусу лоту — лише продажі. Списання вже враховані в загальному складі, але статус лоту відображає реалізацію.
