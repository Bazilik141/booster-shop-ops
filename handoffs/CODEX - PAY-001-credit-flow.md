# CODEX HANDOFF — PAY-001 · Оплата частинами (кредитний флоу товарної сторінки + чекаут)

## 1. Task ID
PAY-001 (ідентифікатор прототипу; формального roadmap Task ID не надано — перед виконанням зареєструвати завдання в роадмапі під цим або погодженим ID).

## 2. Контекст
На проді зараз: клік на тип кредиту в картці товару відкриває модалку "Виберіть кредитну пропозицію" зі списком банків (reference-скріншот від власника — Comfy-подібна модалка з сегментованим вибором кількості місяців 3/5/7/10, чекбоксом "Перший внесок", деталями суми). Блок "Купити в кредит" на товарній сторінці по суті дублює цю модалку.

Дизайн-прототип (design tool project, файли `PAY-001 Купити в кредит.html` + `pay001-credit.jsx` + `pay001-checkout.jsx` + `pay001-screens.jsx` + доповнення в `shared-ui.jsx`) перепроєктовує весь флоу. Це UI/UX-специфікація — не production-код; OpenCart/Twig/PHP реалізація має бути написана Codex проти реального бекенду, з точним відтворенням описаної тут поведінки, копірайту та значень.

Бренд-контекст: додані офіційні гайдлайн-матеріали monobank (лапка, кольори, шрифт Exo 2 для цифр у лапці) — лапка в прототипі намальована власноруч inline SVG (наближення), а не офіційний вектор бренду.

## 3. Мета
1. Товарна сторінка: замінити активну кнопку "Купити в кредит" на окрему кнопку **"Оплатити частинами"** (вторинна відносно "Додати в кошик", але добре видима) — вона відкриває модалку вибору кредитної пропозиції.
2. Блок "Купити в кредит" перетворити на **суто інформаційний** (не клікабельний): назва банку, "До 5 платежів", лейбл "Покупка частинами", лого банку.
3. Модалка "Виберіть кредитну пропозицію": для monobank додати перемикач кількості платежів (3/4/5), живий перерахунок щомісячного платежу, прибрати опцію "перший внесок" (бо перший платіж списується одразу при покупці), показувати лише: щомісячний платіж, кількість платежів, вартість товару. Розширені деталі — на чекауті.
4. Чекаут: пункт оплати "Оплатити частинами" при виборі розгортає **список підключених кредитів** — сьогодні тільки monobank (робочий, з тим самим перемикачем 3/4/5, попередньо обраним на товарній сторінці) + ПУМБ (окрема картка-плейсхолдер, неактивна зона, тег "Скоро буде").
5. Прибрати напис **"Без першого платежу"** повністю — і на товарній сторінці, і в чекауті.
6. Кольори monobank — чорний/білий (бренд-гайдлайн), без синього; ПУМБ — власний брендбук (червоний фон, білий текст) буде опрацьовано окремим раундом; зараз лише коректне лого + копірайт.
7. Трастлайн (Гарантія оригінальності / Швидка відправка / Telegram підтримка) — 3 рівні картки з іконкою над підписом (не 4 колонки з розділювачами), позиціонується під новим блоком кредитування.

## 4. Що змінити (по компонентах)

**Товарна сторінка, права колонка (порядок зверху вниз після ціни):**
- Кнопка "Додати в кошик" — без змін.
- Нова кнопка **"Оплатити частинами"**: текст без іконки (спільна кнопка для всіх банків, не лише monobank), стиль — білий фон, чорна 1.5px рамка, чорний текст (`bs-btn`, не `bs-btn-primary`), висота узгоджена з рештою кнопок. `onClick` відкриває модалку вибору кредитної пропозиції. Приховується, якщо ціна товару < 500 ₴.
- Інформаційний блок "Оплата частинами" (раніше "Купити в кредит"): картка (не кнопка, без courser/hover-афордансу), містить:
  - Рядок monobank: лапка-іконка (26px) + текст "Покупка частинами monobank" + підпис "До 5 платежів".
  - Рядок ПУМБ: лого ПУМБ (реальний асет — див. п.9 нижче) + текст "Оплата частинами ПУМБ" + маленький тег "Скоро буде" (uppercase, сірий пігулка-бейдж) + підпис "До 5 платежів" (раніше було "до 4 платежів без відсотків" — вирівняно з monobank).
  - Приховується разом з кнопкою при ціні < 500 ₴.
- Трастлайн: 3 картки в ряд (не 4), кожна — окрема біло-бордерна плитка з іконкою над підписом, по центру: "Гарантія оригінальності" (щит), "Швидка відправка" (блискавка), "Telegram підтримка" (літак).

**Модалка "Виберіть кредитну пропозицію":**
- Картка monobank: лапка (30px) + wordmark "monobank" + підпис "Покупка частинами"; під ним — перемикач кількості платежів, 3 пігулки: "3 платежі" / "4 платежі" / "5 платежів" (активна — чорна заливка, неактивні — біла з сірою рамкою); нижче — 3 показники в ряд: "Щомісячний платіж" (₴, = округлення вгору від ціна/кількість), "Платежів" (обране число), "Вартість товару" (повна ціна); кнопка "Обрати" (чорна заливка) — переносить обране число платежів у чекаут і одразу вибирає спосіб оплати "Оплатити частинами".
- Картка ПУМБ: лого + "Оплата частинами ПУМБ" + тег "Скоро буде" + "До 5 платежів"; вся картка неактивна (без кнопки вибору).
- Прибрано: чекбокс "перший внесок", розгорнуті "Сума в кредит / Перший платіж / Платежів до завершення" (це тепер лише на чекауті).
- Один дисклеймер-рядок знизу: "Без комісії для вас — умови кредитування визначає банк."

**Чекаут, спосіб оплати:**
- Пункт списку способів оплати перейменовано з "Покупка Частинами monobank" на **"Оплатити частинами"** / підпис "Розстрочка від підключених банків" (без бейджа "Без першого платежу").
- При виборі цього пункту розгортається "шухляда" (той самий блакитний tint, що й обраний радіо-рядок; верхня межа шухляди = відсутня нижня межа радіо-рядка вище — рамка має читатися як одна суцільна форма, не обриватися):
  - Картка monobank (власна біла рамка всередині шухляди): чорний "стікер"-бейдж (заокруглена чорна пігулка з контурною білою лапкою + "monobank" білим) + той самий перемикач 3/4/5 (попередньо обраний з товарної сторінки, редагований); нижче — 3 показники: "Сума в кредит" (₴ ціна), "Щомісячний платіж" (₴), "Платежів до завершення" (обране число − 1); рядок "Кредит буде оформлено на номер телефону: +38 099 111 22 33" (у проді — реальний номер отримувача з форми, не хардкод).
  - Картка ПУМБ (окрема, під нею, пунктирна рамка, приглушена): лого + "Оплата частинами ПУМБ" + тег "Скоро буде" + "До 5 платежів"; повністю неактивна (без чекбоксів/перемикачів).

**Копірайт, що видаляється всюди:** "Без першого платежу" (товарна сторінка + модалка + чекаут).

## 5. Не чіпати
- `sitemap.xml`, `robots.txt`, редіректи, canonical, `.htaccess`.
- Схема/структуровані дані товару, Merchant-фід, GTIN/reviews.
- Логіка кошика, розрахунок цін/знижок, доставка (Нова пошта тощо) — крім суто текстового підпису номера телефону в блоці кредиту.
- Способи оплати "Картка", "Оплата при отриманні", "IBAN" — лишаються без змін (тільки сусідній пункт "Оплатити частинами" редизайнується).
- Реальна інтеграція з API monobank "Покупка частинами" (webhook підтвердження, реальний розрахунок ставки/умов) — **поза межами цього патча**, це суто UI-шар; бекенд-логіка кредитного провайдера — окреме завдання.
- Бренд ПУМБ (червоний фон/білий текст, офіційні промо-матеріали) — окремий раунд, зараз лише лого + копірайт.
- Мінімальна сума кредиту (500 ₴) та максимальна кількість платежів (5) — це прототипні значення для демонстрації; **не вважати їх остаточними бізнес-правилами** без підтвердження від договору з monobank/ПУМБ.

- Модалка кредитної пропозиції — окремий JS/Twig-партіал (назва невідома, шукати за поточним копірайтом "Виберіть кредитну пропозицію").
- Чекаут — партіал способів оплати (шукати за "Покупка Частинами" / "Покупка частинами").

Codex повинен підтвердити ці шляхи в реальному репозиторії перед правками — вважати список орієнтовним.

## 7. Критерії приймання (вимірювані)
- На товарній сторінці при ціні ≥ 500 ₴ видно ДВІ окремі речі: кнопку "Оплатити частинами" (без іконки лапки) і нижче — неклікабельну картку з рядками monobank ("До 5 платежів") та ПУМБ ("Оплата частинами ПУМБ", тег "Скоро буде", "До 5 платежів").
- При ціні < 500 ₴ обидва елементи відсутні в DOM (як зараз).
- Клік на "Оплатити частинами" відкриває модалку; клік деінде на товарній сторінці (не на кредитному блоці) її не відкриває.
- В модалці перемикання 3→4→5 миттєво змінює число "Щомісячний платіж" на `Math.ceil(price / count)` без перезавантаження.
- Текст "Без першого платежу" відсутній у вихідному HTML на всіх трьох екранах (перевірка через пошук по тексту).
- У чекауті вибір "Оплатити частинами" показує рамку без візуального розриву між радіо-рядком і шухлядою (суцільна межа з одним кольором/товщиною), і ВСЕРЕДИНІ шухляди monobank і ПУМБ — це дві окремі картки з власними рамками (не одна вкладена в іншу).
- Кількість платежів, обрана в модалці на товарній сторінці, збігається з попередньо обраною в чекауті (той самий стан, не скидається).
- Трастлайн — 3 картки в ряд, не 4.
- Кольори monobank-елементів — чорний/білий, без синього фону/бейджів.

## 8. QA / smoke-тест
Оскільки патч торкається чекауту й способу оплати — **обов'язково прогнати `bs-checkout-smoke`** (11-крокова ручна перевірка чекаут-флоу) після виконання, перш ніж рахувати завдання завершеним.

Додатково для цього патча вручну перевірити:
- Мобільна ширина (390px): кнопка "Оплатити частинами" і картка кредиту не ламають вертикальний рітм; модалка відкривається як bottom sheet на всю ширину колонки (не на всю ширину реального вікна браузера).
- Перемикач 3/4/5 у чекауті на вузькій ширині не обрізається і не перекриває сусідні елементи (flex-wrap).
- Всі 4 сценарії відмови (клієнта не знайдено / недостатньо коштів / клієнт відхилив / час очікування) досі показують коректний текст після перейменування "Покупка Частинами monobank" → "Оплатити частинами" (текст відмов посилається на "застосунок monobank", не на назву способу оплати — конфлікту немає, але перевірити візуально).

## 9. Активи, які потребують заміни перед продом
- **Лапка monobank**: у прототипі — намальована вручну inline SVG (наближення форми, чорна заливка + білий контур + тінь). Перед продом замінити на офіційний векторний асет із бренд-кіту monobank (Codex/дизайнер має завантажити з офіційних гайдлайн-матеріалів, не використовувати SVG-наближення з прототипу).
- **Лого ПУМБ**: у прототипі використано зображення, завантажене власником у design-tool проєкт (`uploads/pasted-1784456162944-0.png`, 160×160 PNG з прозорими кутами). Це тимчасовий шлях, специфічний для дизайн-інструменту — для продакшена потрібен постійний, коректно ліцензований асет під звичайним шляхом бренд-асетів сайту (наприклад `catalog/view/image/payment/pumb-logo.png`), а не hotlink на файл дизайн-проєкту.
- Шрифт Exo 2 (для цифр усередині лапки за гайдлайном) у самому UI-патчі **не використовується** (цифри платежів показані звичайним текстом поруч, не всередині графічної лапки) — підключати цей шрифт для цього патча не потрібно.

## 10. Rollback
Патч суто presentation-layer: жодних змін схеми БД, жодної нової бекенд-логіки платежів (реальна інтеграція monobank API — окреме завдання, тут не чіпається). Rollback = відкат зачеплених шаблонів/партіалів і CSS-файлу до бекапу, знятого безпосередньо перед деплоєм цього патча. Дані замовлень/платежів не зачіпаються — відкат бек-апу даних не потрібен.
